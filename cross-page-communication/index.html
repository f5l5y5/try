前端跨页面通讯终极指南

9种跨页面通讯方式的实践案例和详细说明

什么是跨页面通讯？

跨页面通讯是指在浏览器中不同页面之间传递数据和消息的技术。在现代 Web 应用中，跨页面通讯是实现多标签页协同、状态同步、实时通知等功能的重要基础。

本指南收集了9种常用的前端跨页面通讯方式，包括它们的基本原理、使用方法、优缺点和实践案例。每种方式都有完整的代码示例和详细说明，方便你学习和使用。

1. postMessage

HTML5 引入的 API，允许来自不同源的脚本采用异步方式进行有限的通信，可以实现跨窗口、跨域的消息传递。

核心特性

  • 支持跨域通信
  • 双向通信
  • 异步操作
  • 浏览器兼容性好

2. BroadcastChannel

HTML5 引入的 API，允许同源下的不同浏览器上下文之间进行简单的广播通信。

核心特性

  • 自动关联相同名称的频道
  • 广播机制
  • 简单易用
  • 异步通信
查看示例 查看文档

3. MessageChannel

HTML5 引入的 API，用于创建一个双向通信通道，允许两个不同的执行上下文之间进行高效、安全的通信。

核心特性

  • 双向通道
  • 端口传递
  • 高效传输
  • 支持 Transferable 对象
查看示例 查看文档

4. localStorage

HTML5 引入的 Web Storage API 的一部分，用于在浏览器中存储键值对数据，通过监听 storage 事件实现跨页面通信。

核心特性

  • 持久存储
  • 简单易用
  • 事件驱动
  • 浏览器兼容性好
查看示例 查看文档

5. SharedWorker

Web Workers API 的一部分，用于创建可以被多个同源页面共享的后台线程，实现真正的跨页面通信。

核心特性

  • 多页面共享
  • 后台运行
  • 持久存在
  • 支持点对点和广播消息
查看示例 查看文档

6. window.name

浏览器窗口的一个属性，用于获取或设置窗口的名称。在页面导航时，window.name 属性的值会保持不变，可以用于跨页面传递数据。

核心特性

  • 持久性
  • 跨域支持
  • 简单易用
  • 大容量
查看示例 查看文档

7. ServiceWorker

浏览器在后台运行的脚本，独立于网页，用于增强网页的功能。可以拦截网络请求、管理缓存、推送通知等，也可以用于实现跨页面通讯。

核心特性

  • 后台运行
  • 网络拦截
  • 缓存管理
  • 推送通知
查看示例 查看文档

8. indexDB

浏览器提供的本地数据库，用于存储大量结构化数据。通过监听数据库变化实现跨页面数据同步。

核心特性

  • 大容量存储
  • 结构化数据
  • 事务支持
  • 索引支持
查看示例 查看文档

9. Cookie 和 Clipboard 轮询

通过定期检查 Cookie 或剪贴板的变化来实现跨页面通信。Cookie 轮询简单易用，Clipboard 轮询支持大容量数据。

核心特性

  • Cookie 轮询：简单易用，兼容性好
  • Clipboard 轮询：大容量，近实时
  • 需要定期检查
  • 实现简单
